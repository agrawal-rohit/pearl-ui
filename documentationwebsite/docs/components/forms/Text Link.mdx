The `TextLink` component in Pearl UI is a versatile and customizable element that combines a [Text](../typography/Text) component with a [Pressable](./Pressable) component. It can be used to trigger various actions or events within your application, such as submitting a form, opening a dialog, canceling an action, or performing a delete operation. By default, it renders the [Pressable](./Pressable) component.

## Import

```jsx
import { TextLink } from "pearl-ui";
```

## Usage

```jsx
<TextLink>I'm a pressable text</TextLink>
```

### TextLink sizes

The `size` prop allows you to adjust the size of the text link. The value for this prop should be one of the keys available in <t>PearlTheme.components.TextLink["sizes"]</t>, which are <t>"xs"</t>, <t>"s"</t>, <t>"m"</t>, and <t>"l"</t> in the [default theme](#default-component-config).

```jsx
<TextLink size="xs">Tiny Pressable Text</TextLink>

<TextLink size="s">Small Pressable Text</TextLink>

<TextLink size="m">Regular Pressable Text</TextLink>

<TextLink size="l">Large Pressable Text</TextLink>
```

### TextLink color scheme

The `colorScheme` prop allows you to change the color palette of the text link. This is a powerful feature as it changes all the use color values in a palette through a single prop. The value for this prop should be one of the keys available in <t>PearlTheme.palette</t> that contain a palette of colors represented as an object, which are <t>"primary"</t>, <t>"secondary"</t>, <t>"neutral"</t>, etc in the [default Pearl theme](../../theming/default-theme#color-palette).

```jsx
<TextLink colorScheme="primary">Primary Pressable Text</TextLink>

<TextLink colorScheme="secondary">Secondary Pressable Text</TextLink>
```

### Override Style

The `TextLink` component supports a wide range of style props which can be used to override the pre-defined variant style in the theme. These manual style props have higher precedence than the [default component config](#default-component-config), allowing for greater customization.

```jsx
// passing style prop m="xl" overrides the default component config value
<TextLink m="xl" />
```

## Accessibility

- `TextLink` has the `role` of `button`.
- `TextLink` has the default accessibility label set as text value passed into it. A custom label can be specified using the `accessibilityLabel` prop.
- When `TextLink` is disabled or loading, it is reflected as `disabled` and `busy` respectively in screen readers.
- Similar to [Pressable](./Pressable), `TextLink` expects an `actionDescription` prop to specify the intented outcome of interacting with the component eg. 'Closes modal', 'Go to the homepage', etc.

## Component Properties

### Supported Style Properties

The `TextLink` component is a composition of the [Pressable](./Pressable) component. Therefore, all properties related to the [Pressable](./Pressable) component can be passed to the `TextLink` component, in addition to the properties listed below.

### Additional Properties

| Name          | Required | Type                                              | Default     | Description                                     |
| ------------- | -------- | ------------------------------------------------- | ----------- | ----------------------------------------------- |
| `size`        | No       | <t>PearlTheme.components.TextLink["sizes"]</t>    | `"m"`       | Defines the size of the text link.              |
| `variant`     | No       | <t>PearlTheme.components.TextLink["variants"]</t> |             | Specifies the variant of the text link.         |
| `colorScheme` | No       | <t>PearlTheme["palette"]</t>                      | `"primary"` | Sets the active color palette of the text link. |
